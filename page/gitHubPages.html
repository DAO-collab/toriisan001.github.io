gitHubPages

概要

GitHub Pagesでウェブサイトを公開しよう

ユーザーページとプロジェクトページがあるらしい

ユーザーページ

https://github.com/toriisan001/toriisan001.github.io

https://toriisan001.github.io

プロジェクトページ

https://github.com/toriisan001/MineSweeper

https://toriisan001.github.io/MineSweeper


ユーザーページは1アカウント1つまで、プロジェクトページは複数作ることが出来るらしい

アカウントの作成

https://github.comでアカウントの作成

リポジトリの作成

https://github.com/new


リポジトリ名「ユーザーネーム + github.io」は固定

>toriisan001.github.io


なお、ユーザーサイトとプロジェクトサイトの違いはこちらのリポジトリ名の指定だけになる。

ユーザーサイトの場合はルールがありましたが、プロジェクトサイトの場合は自由に名前を決めることができる。

更新の仕方

「<>Code」タブでAdd Fileから「Upload files」ファイルをアップロードして「Commit Change」


「<>Code」タブ「・・・」Setting下のほうの

Select BranghをNoneからmainに修正

rootのままSaveを選択

※ 反映が遅い場合

Environments - github-pagesから最新の履歴の更新idを確認

urlの末尾にidを付けてアクセスすると何故か反映された。

例：「https://toriisan001.github.io/?dcdcc96」

リポジトリの削除

「<>Code」タブ「・・・」Setting下のほうの「Delete this repository」

文字列を書き込んで削除ボタンを押す

環境にアクセス

Environments - github-pages を押して「View deployment」を選択

ローカルにクローン。コミットプッシュ

「<>Code」タブ、緑の「Code▼」ボタンからHttpsの内容をコピー

右クリック、git clone。URLに上記内容をコピーしてOK「toriisan001.github.io」ディレクトリが出来る。中に「.git」ファイルも出来ている。

削除したいときはこのディレクトリごと削除すればいいらしい。

更新後、右クリック。「Git Commit -> main」メッセージを追加して「Commit」でコミット

「toriisan001.github.io」ディレクトリを右クリック。TortosGit→push。OKをクリック

ログインした時のメールアドレス。パスワード、ユーザー名。Personal access tokens (classic)を入力するとpushが完了する。以下参照

【Windows】Gitでリモートにアクセスができない時の解決方法（Support for password authentication was removed on August 13, 2021.）

新しいPCに環境をclone(既存のリポジトリをローカル環境に複製)

詳細

VSCodeのダウンロードページ

Git_for_Windowsをダウンロード・インストール

TortoiseGitをダウンロード・インストール

Cドライブ直下に作業用ディレクトリ「workspace」を作成。

自分のGitHabPagesのGitURLを確認。

ディレクトリ「C:\workspace\toriisan001.github.io」で右クリック → 「Clone...」をクリック。確認した「Git URL」を貼り付けて「OK」をクリック。

「close」をクリック。

Nodejsのインストーラをダウンロード・インストール

Nodejsのダウンロードページ

Cmdを起動して作業用ディレクトリに移動する。「cd C:\workspace\toriisan001.github.io」

「npm install -g http-server」 で http-server をインストール

「npm install -g reload」 でホットリロードするパッケージをインストール

「reload -b」 で1分ごとにホットリロードで立ち上げ

ブラウザを起動して「http://localhost:8080/」をurlに入力すればアクセス出来る。
