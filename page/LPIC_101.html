コマンド

init

init 6

ランレベル「6」はシステムの再起動を意味します。

init 0

ランレベル「0」はシステムの停止を意味します。

init 1

「init 1」はシングルユーザーモードでの起動を実施します。シングルユーザーモードは root のみがログインできるモードです。またネットワークアクセスは行えないため、該当のマシン上からしか操作が行えません。

init 2

「init 2」はネットワーク無しのマルチユーザーモードでの起動を実施します。ネットワーク無しのマルチユーザーモードは、複数ユーザーがログインできますが、ネットワークアクセスは行えないため、該当のマシン上からしか操作が行えません。

init 3

「init 3」は通常のマルチユーザーモードでの起動を実施します。通常のマルチユーザーモードでは、Telnet や SSH など、ネットワークを介した、複数ユーザーでのログインが行えますが、グラフィカルユーザーインターフェース（Xディスプレイマネージャ）は起動できません。

init 5

「init 5」はグラフィカルユーザーインターフェース（Xディスプレイマネージャ）での起動を実施します。

IRQ 【Interrupt Request】

コンピュータ内部の装置や周辺機器などが中央処理装置（CPU）に信号を送り、現在の処理を中断して強制的に指定した処理を実行するよう要求すること。「割り込み要求」とも訳される。

I/Oアドレス

パソコンの回路や周辺機器などをCPUが識別するための番号。 パソコン内部のデータの受け渡しに使われている。

DMA (Direct Memory Access)

プログラムされた機械語の命令群の実行によってアキュムレータなどを介する方法によらず、メモリとメモリまたはメモリとI/Oデバイスの間で直接データを転送することである。 専用回路のことを DMACと言う。

アキュムレータ:日本語で「蓄圧器」と言い、電気で言う「充電式バッテリー」と似た働きをします。 電気においては蓄電池に電気のエネルギーを蓄えますが、液圧においては液体のエネルギー(液体の圧力）をアキュムレータに蓄えておき、必要に応じてそのエネルギーを放出します。

階層型に設計されたオペレーティングシステム の中核となる部分で、アプリケーションとハードウェアの架け橋である。具体的には、システムのリソースや、ハードウェアとソフトウェアの連携を管理する。その他、通信制御も行う事が多い。

ソフトリンク は、コンピュータのディスク上で扱うファイルやディレクトリを、本来の位置にファイルを残しつつそれとは別の場所に置いたり別名を付けてアクセスする手段である。複製とは違い、実体がないこと、ソフトリンクで開いたファイルへの操作が実物のファイルにも反映されること、ファイルサイズが小さいのが特徴。

ハードリンク

ファイルの実体を直接参照するリンクのことです。同じ実体を参照するハードリンクは同じinode番号を持ち、複数作成してもコピーのようにディスク容量を消費しません。そのため、更新が無いファイルのバックアップ手段としてハードリンクが使用されることもあります。また、ハードリンク先のファイルの中身を編集すると、元ファイルの中身も同じく変更されます。

管理上“別の名前で参照している”ということを分かりやすくしておきたい場合はシンボリックリンク、区別をしたくないときはハードリンク、という使い分けができます

ハードリンクには、「（1）同じパーティション内にないといけない」「（2）ディレクリへのハードリンクはスーパーユーザーにしか作れない（さらに「-d」オプションが必要）」という制限があります。

【参考】【 ln 】コマンド――ファイルのハードリンクとシンボリックリンクを作る

inode番号はファイルシステム毎に管理されるため、異なるファイルシステムへハードリンクを作成する事はできません。

ディレクトリのハードリンクは作成できません。

ハードリンクを作成

「ln ファイル名 リンク名」(lnは小文字のLと小文字のn)

[root@localhost ~]# vi sample.txt

[root@localhost ~]# cat sample.txt

[root@localhost ~]# ln sample.txt sample_hard
[root@localhost ~]# ls
anaconda-ks.cfg  sample_hard  sample.txt
[root@localhost ~]# ls -li

34367820 -rw-------. 1 root root 1431 16. Jul 2015  anaconda-ks.cfg
34128087 -rw-r--r--  2 root root    7  1. Jul 12:13 sample_hard
34128087 -rw-r--r--  2 root root    7  1. Jul 12:13 sample.txt
※inode番号34128087を見るとハードリンクの数が2になっていることが確認できる

シンボリックリンク

Windowsでのショートカットのようなもので、元ファイルの場所を指し示すリンクのことです。シンボリックリンクが持っている情報は「元ファイル（ディレクトリ）がどこにあるのか」というパス情報のみです。シンボリックリンクの元ファイルを削除すると（他に同じinode番号を持つハードリンクが無ければ）、ファイルの実体は削除されますが、リンクは残ります。ただその場合、シンボリックリンクにアクセスすると、リンクしている元ファイルが無いためエラーになります。

シンボリックリンクを作成

ln -s ファイル名 リンク名

[root@localhost ~]# ls
anaconda-ks.cfg  sample.txt
[root@localhost ~]# ln -s sample.txt sample_symbolic
[root@localhost ~]# ls -li
insgesamt 8
34367820 -rw-------. 1 root root 1431 16. Jul 2015  anaconda-ks.cfg
34128086 lrwxrwxrwx  1 root root   10  1. Jul 12:23 sample_symbolic -> sample.txt
34128087 -rw-r--r--  1 root root    7  1. Jul 12:13 sample.txt
※inode番号34128086の先頭にシンボリックリンクを示す「l」がパーミッションの先頭に表示されています。
※inode番号34128086のファイル名を示す欄では、sample_symbolicはsample.txtのシンボリックであることが分かります。矢印が付いている。

inode番号

index nodeの略で、Linuxなどのファイルシステムで使用されているファイルなどについての情報を記した管理データのことを指します。inodeはファイルやディレクトリごとに作成されて、それぞれファイルやディレクトリの作成時に一意のinode番号が割り当てられます。inodeで記録されている情報は、パーミッションや所有者、ファイルサイズなどがあります。inodeはファイルシステム作成時に最大数を決定するのでinodeが枯渇するとディスクに空き容量があってもファイルが作成できなくなる可能性があります。

「ls -li」(-iオプション)でファイルのinode番号で確認できる。一番左の番号がinode番号

パーティションの設計について

/varは高速に書き込みできるディスクに割り当てる。書き込みが頻繁に発生するため

パーティション:とはLinuxやWindowsなどに関係なく一般的な仕組みの一つで、PCの記憶媒体、HDDやSSDその他メモリスティックなどにもうけることが出来る「しきり」のこと

システムがRAID上に構成されている場合、カーネルがブートローダから参照できる場所に配置されている必要があるため、/bootだけをRAIDに含めない構成を取る場合がある。

RAID(Redundant Array of Inexpensive Disks（リダンダント・アレイ・オブ・インエクスペンシブ・ディスクズ）):「安価なディスクを使った冗長配列」となります。複数の外部記憶装置（ハードディスクなど）を一台の記憶装置として冗長運用（システムを止めずに運用）する技術のこと

ルートパーティション:ルートディレクトリを含むディスクパーティションを指すUNIX用語。

任意のパーティション（またはファイル）にスワップ領域を作成できます。

スワップ領域:LinuxなどのUNIX系OSでストレージ上に作成される特殊なパーティションの一つで、メインメモリ（物理メモリ）の容量の延長としてプログラムやデータを記録することができる領域。
