命名規則

キャメルケース（英: camel case）

「hogeData」や「HogeData」のように単語の先頭を大文字にしてつなげる表記方法

プログラミングでの関数や変数

関数名は先頭小文字。例:createBlogCategory()

変数名(引数名)は先頭小文字。例:companyId (先頭の単語だけ小文字にするのがローワーキャメルケースと呼ぶ)

class名は先頭大文字。例:BlogCategory (先頭の単語が大文字をパスカルケースまたはアッパーキャメルケースと呼ぶ)

スネークケース（英: snake case）

「hoge_data」のように単語間をアンダースコアでつなげる表記方法

DBなどで扱う値

定数はすべて大文字。例:BLOG_CATEGORY

ケバブケース（英: snake case）

「hoge-hoge」単語の間をハイフンでつなぐ命名規則です。HTMLではclass名であったり、ファイル名などでもよく見かけます。

HTTPメソッド(ブラウザ)

ブラウザの制約上、実際に使用されるのは、GET、POSTの2種類らしい。

GET

データベースの書き換えが必要なく、データを取得するだけの場合に使用。

クエリパラメータでデータを受け渡す。(URL?パラメータ名1=値1&パラメータ名2=値2&)

POST

メッセージボディでデータを受け渡す。URLにパラメータがつかない。

データを変更したい時に使用。パスワードのような秘匿情報をURLに表示したくない場合。データ容量が多い場合に使用

HTTPメソッド(API)

REST(REpresentational State Transfer) API「シンプルなWebシステムの設計思想」があるらしい

GET 、 POST 、 PUT 、 PATCH 、 DELETE が用意されているらしい。

同じURL(URLにリソースを書く)でHTTPメソッドだけを変えて、リソースに対する操作を行う

GET

取得

POST

リソースの作成

PUT

データが存在しているかどうかわからないときに使用します。データが存在しているときは更新をし、データが存在しない場合は新規作成を行う

PATCH

データがすでに存在しているものに対して更新をかける処理

DELETE

削除

HTMLタグの使い分けについて

<!DOCTYPE html>

タグではない

HTML 文書ファイルの先頭に記述し、その HTML ファイルで使用している HTML のバージョンを宣言している

htmlタグ

ドキュメントがHTML文書であることを示すタグ

lang属性で言語を指定している

headタグ

metaタグ・titleタグ・linkタグを持つ

metaタグ

Webサイトの情報を記載する

titleタグ

タイトルを設定できる。ブラウザのタブに表示される

linkタグ

rel属性にiconを設定してファビコン(ブラウザのタブの画像)が設定可能

rel属性にstylesheetを指定してcssを読み込み可能

bodyタグ

文章や画像など、実際にブラウザの画面上に表示される内容を指定するタグ

文書や画像、リンク等を指定できる様々なタグを持つ

headerタグ

文書やセクションのヘッダーを表します。文書やセクションの冒頭となる見出しや概要、ナビゲーションのリンクなどを記述する場合によく利用される

sectionタグ

文書やアプリケーションの一部分となる、意味や機能のひとまとまりのこと

その範囲の内容に明確なテーマがあればsectionを用い、特にテーマなどがなく汎用的に範囲指定する場合divを用いるという判断が良いらしい

mainタグ

Internet Explorer(IE)だけは使うことができないらしい

mainタグにはナビゲーションやフッター情報などを除いたWebサイトのメインとなるコンテンツを記載するらしい

mainタグは各ページで一度しか使うことができないらしい

ただ記事を分けたいだけなら汎用的なdiv要素を使った方が良いらしい

h1～h6タグ

hというのはHeading（見出し）の意味

h2からh6は何回使っても問題ない。h1は最上位の大見出しとなるので原則1ページ1回の使用が推奨らしい。使ってもエラーにはならないが

divタグ

各コンテンツをひとまとめする（グループ化する）際に使う汎用的なタグ

便利だが使いすぎるとブラウザの技術が各要素を特定するのに時間がかかるので、Webページの読み込みの速度が遅くなるらしい

spanタグ

囲んだ要素だけデザインを変更する時に使う

改行しない。「display: block; 」を適用すると改行するようになるらしい

pタグ

「P」とは「Paragraph」の略

pタグで囲まれたテキスト1つの段落であることを示す

長すぎる場合は、brタグを使って改行すると良いらしい

preタグ(preformatted text（整形済みテキスト）)

整形済みであることを示す。改行などもそのまま表示される。

codeタグ

コードであることを示す

複数行にわたる場合は、preタグも合わせて使用する。

【ソースコードを載せるためのHTMLタグ】codeタグで綺麗なコードを表示しよう

imgタグ

文書内に画像を表示するためのタグ

OL(ordered list)タグ

順序のある番号付きのリストを表示する際に使用するタグ「1,2,3・・・」と番号を振る

UL(unordered list)タグ

順序がない箇条書きのリストを表示する際に使用するタグ「・,・,・,・・・」と点を振る」

LI(list item)タグ

リストの項目を表示するために使用するタグ。OLタグ、ULタグの子要素に配置して使用する

a(anchor（アンカー）)タグ

リンクの出発点や到達点を指定するタグ。href属性でリンク先を指定

クロノドライブのHTML辞典<a>

imgタグ

文書内に画像を表示するためのタグ

brタグ

「<br>」というように終わりにスラッシュ無しで書くのが良いらしい。pタグの文章の中で改行するために使用

fooderタグ

フッター・文書ページの下部に表示されるもの

会社名やメニュー、コピーライト表記など

他のサイトへのリンクを置くことで、他のページへの案内を行うこともあるらしい
